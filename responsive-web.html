Chapter 1: Our Responsive Web

We should embrace the flexibility and variablity inherent to the web, and build sites that adapt to the media that renders them. We need to let go of our desire to control the medium in which audiences view our webpages and creating pixel-by-pixel perfect designs, but rather ensure fluidity in our approach such that any device can satisfactorily view our webpages. Together with utilising standards-based technologies to ensure consistent implementation across browsers, responsive web design consists of three ingredients:

    • A flexible, grid-based layout
      Flexible images and media, and
      Media queries, a module from the CSS3 specification

    Firstly, note that browsers have default styles for HTML elements. So we should use a "reset stylesheet" to override these defaults and create a consistent baseline for all browsers to reduce browser inconsistencies in things like default line heights, margins and font sizes of headings, and so on. We have used Eric Meyer's reset in our stylesheet.

    Proportionate Fonts: To create proportionate fonts that scale with the screen size, we calcuate its relative type size. For example, if we have designed a page (in a fixed form dimension such as A4) to have a header font of 24px, we call this size the "target" size. The baseline type size for most browser defaults (at 100%) is 16px, this gives us the "context" size. We calculate our relative type size ratio as target / context, which will be 24 / 16 = 1.5 in our example.

       h1 {
        font-size: 1.5em; /* 24px / 16px */
        font-style: italic;
        font-weight: normal;
      }
